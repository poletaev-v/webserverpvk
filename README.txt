#### ПРИМЕЧАНИЕ ####
- Программа является HTTP/TCP - сервером, который принимает входящие запросы из той же локальной сети в которой запущен сервер.
- Для проверки программы нужно отправить запрос к фунционирующему серверу с данными из файла "XML".
- Для выхода из программы следует использовать комбинацию клавиш "CTRL+c".
- Конфигурационные данные программы с пояснениями будут описаны в этом документе.
- ВНИМАНИЕ! Конфигурационный файл находится по пути "configs/main.yml", ни в коем случаем нельзя перемещать каталог и сам конфигурационный файл!

#### ПОРЯДОК РАБОТЫ ####
- Сервер принимает входящие запросы из той же локальной сети, в которой запущен сам сервер.
- При поступлении запроса принимаются входные данные содержимого XML файла.
- На экран выводятся те данные, которые указаны в конфигурационном файле "main.yml".
- При поступлении запроса и вывода на экран данных, по истечению заданного (в конфигурационном файле "main.yml") промежутка времени отображается задданое по умолчанию сообщение.

#### Конфигурационные данные с пояснениями разделенные знаком '#', файл "main.yml" ####
#### Данные которые обязательны для изменения помечены знаком '***' ####
http:
  addr: "192.168.0.2" # Локальный адрес сети ***
  port: 80            # Локальный порт HTTP
  maxHeaderMIB: 1     # Максимальный принимаемый заголовок в мегабайтах
  readTimeout: 10s    # Время ожидания для чтения данных
  writeTimeout: 10s   # Время ожидания для записи данных
  refreshURL: "VcjZt3ZMwbmzvtif3Q9WzrjhZnZefctK5Cxw0N5vzSrl5HTtrMMWTTEIHAQbh7TThzyBgUf6lNMdU6VMtKVq2TamtDBpsvcEOFggFnE5UXFZ3sWGHIxMpPGO" # Ссылка для принятия данных от TCP клиента

tcp:
  addr: "192.168.0.2" # Локальный адрес сети *** (является тем же самым параметром, что и локальный адрес сети для HTTP)
  port: 4545          # Локальный порт TCP
  bufLimitMIB: 2      # Максимально отправляемый буффер данных
  awaitConn: 10s      # Ожидание принятия данных от клиента после его подключения к серверу

data:
  timeDuration: 20    # Время в секундах, после которого обновить информацию на экране после принятия данных
  availableFields:    # Данные для показа на экран где наименование - это наименование из XML файла, значение - расшифровка
    TrackGrossWeight: "Общая масса, кг"
    TrackWidth: "Ширина, см"
    TrackWHeight: "Высота, см"
    TrackLength: "Длина, см"
  message: "Ведется весовой и габаритный контроль" # Выводимое сообщение по умолчанию, которое отображается пока нет входящих данных
